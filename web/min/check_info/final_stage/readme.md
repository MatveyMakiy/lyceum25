# Финальный этап проекта

На финальном этапе проект был доведён до состояния полноценного веб-приложения. Основной упор был сделан на завершение функциональных требований, исправление логики ленты, добавление административной части, тестов и визуальной доводки.

## Что было сделано

### Лента

Была переработана логика ленты. Теперь она не показывает все публикации подряд. Авторизованный пользователь видит только:

- свои личные публикации;
- публикации групп, в которых он состоит;
- ближайшие мероприятия из своих групп;
- публичные мероприятия от администрации.

Неавторизованный пользователь может открыть ленту, но видит только публичные мероприятия от администрации. Для просмотра публикаций сообществ ему нужно войти в аккаунт.

### Группы

Была доработана система групп:

- создание групп;
- редактирование групп;
- вступление и выход из группы;
- роли внутри группы;
- назначение и снятие модераторов;
- публикации внутри группы;
- групповой чат.

Редактировать группу может только её администратор.

### Публикации

Была доработана работа с публикациями:

- создание личного поста;
- создание поста в группе;
- редактирование своего поста;
- удаление своего поста;
- лайки;
- комментарии;
- удаление своих комментариев.

Посты групп отображаются в ленте только у участников этих групп.

### Мероприятия

В проекте реализованы мероприятия:

- создание мероприятия в группе;
- редактирование мероприятия;
- удаление мероприятия;
- запись на мероприятие;
- отмена записи;
- счётчик участников;
- публичные мероприятия от администрации.

Публичные мероприятия видны даже неавторизованным пользователям в ленте.

### Чаты

Была реализована система сообщений:

- личные чаты между пользователями;
- открытие личного чата из чужого профиля;
- список чатов;
- отправка сообщений;
- групповые чаты сообществ.

Групповой чат доступен только участникам соответствующей группы.

### Админка

Была добавлена глобальная админка. Глобальный администратор может:

- смотреть статистику сайта;
- создавать публичные мероприятия;
- удалять посты;
- удалять комментарии;
- удалять группы;
- удалять мероприятия;
- удалять пользователей;
- назначать других пользователей глобальными администраторами;
- снимать роль глобального администратора.

Также добавлены ограничения: администратор не может удалить самого себя и не может снять роль администратора с самого себя.

### Тема сайта

Была добавлена смена темы сайта:

- светлая тема;
- тёмная тема;
- переключатель темы в сайдбаре;
- сохранение выбранной темы в localStorage.

### Сайдбар

Был исправлен сайдбар. Теперь он не прокручивается вместе со страницей, а остаётся закреплённым слева. Кнопка выхода всегда доступна внизу сайдбара, поэтому пользователю не нужно прокручивать длинные страницы до конца.

### Тесты

Были добавлены тесты для backend и frontend.

Backend-тесты проверяют:

- авторизацию;
- middleware;
- роли групп;
- логику лайков;
- логику комментариев;
- логику мероприятий;
- логику чатов;
- логику ленты;
- админскую логику.

Frontend-тесты проверяют:

- работу localStorage;
- переключение темы;
- API постов;
- API мероприятий;
- API админки;
- компонент карточки поста;
- сайдбар.

Покрытие тестами было проверено через `npm run coverage`.

## Дополнительные возможности

Дополнительно были реализованы:

- тёмная тема;
- групповые чаты;
- публичные мероприятия;

## Что не было реализовано

Некоторые дополнительные требования не были реализованы, так как проект уже закрывает основные критерии:

- PWA;
- Redis-кэширование;
- загрузка файлов;
- полноценные WebSocket-чаты;
- CI/CD;
- интернационализация;
- деплой на сервер.

## Итог

На финальном этапе проект был доведён до рабочего состояния. Основные пользовательские сценарии работают: регистрация, вход, группы, посты, комментарии, лайки, мероприятия, личные сообщения, групповые чаты и админка. Также были добавлены тесты, тёмная тема и улучшения интерфейса.
