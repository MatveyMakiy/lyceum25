Техническое задание: Текстовый калькулятор на русском языке

1. Цель проекта
Создать консольный калькулятор, который принимает математические выражения, записанные словами на русском языке, анализирует их и возвращает числовой результат.

2. Функциональные требования
    2.1. Поддерживаемые числа
        - Диапазон: от -99 до 99 включительно
        - Форма записи: словами в именительном падеже
        - Примеры: "тридцать три", "минус двадцать", "ноль"
        - Дробные числа и числа вне диапазона не поддерживаются
    2.2. Поддерживаемые операции
        - Сложение: "плюс"
        - Вычитание: "минус"
        - Умножение: "умножить на"
        - Деление: "поделить на"
    2.3. Формат выражений
        - Выражение начинается с числа (возможно, с отрицательного знака "минус")
        - За числом следует операция, затем следующее число
        - Может содержать несколько операций подряд
        - Пример: "два плюс три умножить на четыре"
    2.4. Приоритет операций
        - Умножение и деление выполняются перед сложением и вычитанием
        - Нет поддержки скобок в текстовой форме

3. Валидация и обработка ошибок
    3.1. Проверка чисел
        - Корректность записи числа
        - Проверка диапазона (-99 до 99)
        - Распознавание отрицательных чисел ("минус пять")
    3.2. Проверка выражений
        - Корректная последовательность: число-операция-число-...
        - Отсутствие операций без чисел
        - Отсутствие чисел без операций (кроме начала выражения)
    3.3. Обработка ошибок
        - Деление на ноль
        - Некорректный ввод
        - Пустые выражения
        
4. Интерфейс
    4.1. Ввод
        - Выражение вводится одной строкой
        - Регистр не имеет значения
        - Пробелы между словами стандартные
        - Поддерживаются только русские слова
    4.2. Вывод
        - Числовой результат вычисления
        - В случае ошибки — понятное сообщение с описанием проблемы
    4.3. Примеры ввода
        - тридцать три поделить на три
        - два плюс два плюс два  
        - минус два умножить на двадцать
        - девять минус пять плюс один
    4.4. Примеры вывода
        - Результат: 11
        - Результат: 6
        - Результат: -40
        - Результат: 5

5. Дополнительные требования
    5.1. Работа программы
        - Возможность многократного ввода
        - Возможность выхода по команде ("выход", "exit")
        - Очистка ввода от лишних пробелов
    5.2. Обработка особых случаев
        - Число "ноль" во всех операциях
        - Отрицательные числа в начале и середине выражения
        - Многократное использование одной операции

6. Ограничения
    6.1. Не поддерживается
        - Дробные числа
        - Числа больше 99 и меньше -99
        - Сложные составные числа ("сто двадцать три")
        - Степени, корни, проценты
        - Скобки в текстовой форме
        - Пробелы внутри слов чисел
